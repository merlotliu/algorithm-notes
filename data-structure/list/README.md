## 面试&笔试

在面试和笔试中，对算法的要求应有所区分。

在笔试中，题量多时间少，我们要尽量采取写出容易想到并且时间复杂度符合要求的算法，通常可以以空间换时间。

而在面试中的题，通常难度更小，为了给面试官留下深刻的影响，应尽量写出低时间复杂度，低空间复杂度，能体现代码水平的代码。
